## Context

Сейчас `/start` отправляет сообщение с единственной WebApp-кнопкой и текстовым списком команд. Пользователь вынужден либо печатать команды, либо открывать Mini App. Все остальные меню в боте (admin, shop) используют многострочные `InlineKeyboard` через `btnText()`. Нужно привести `/start` к тому же паттерну.

## Goals / Non-Goals

**Goals:**
- Заменить текстовый список команд на инлайн-кнопки в `/start`
- Каждая кнопка открывает соответствующий раздел бота (как если бы пользователь ввёл команду)
- Рефакторинг: вынести тела команд в отдельные функции для переиспользования из callback'ов
- Кнопка «Админка» только для админов
- Сократить текст сообщения `/start` — убрать дублирующийся список команд

**Non-Goals:**
- Не добавлять ReplyKeyboardMarkup
- Не менять содержимое самих разделов (buy, profile, generate, history, support)
- Не менять `/app` команду
- Не менять Mini App

## Decisions

1. **Диспетчеризация: вынос тел команд в функции (вариант B)**
   - Из `bot.command("buy", ...)` выносится тело в `async function showBuy(ctx, db)`, регистрируется и команда, и callback
   - Альтернатива (посылать `/command` через `ctx.api.sendMessage`) отклонена — создаёт лишние сообщения в чате

2. **Формат callback data: `nav:<command>`**
   - Например: `nav:buy`, `nav:generate`, `nav:profile`, `nav:history`, `nav:support`, `nav:admin`
   - Единый хендлер `bot.callbackQuery(/^nav:(\w+)$/)` диспетчеризует по switch

3. **Расположение nav-хендлера**: в `server/bot/index.ts` внутри `createBot()`, после всех `register*` вызовов, чтобы все функции уже были зарегистрированы

4. **Лейаут клавиатуры:**
   ```
   ┌─────────────────────────────────────┐
   │  🚀 Открыть приложение   (WebApp)   │
   ├───────────────────┬─────────────────┤
   │  💎 Купить         │  🎵 Генерация   │
   ├───────────────────┼─────────────────┤
   │  👤 Профиль        │  📜 История     │
   ├───────────────────┴─────────────────┤
   │  ❓ Поддержка                        │
   ├─────────────────────────────────────┤
   │  ⚙️ Админка (только для админов)    │
   └─────────────────────────────────────┘
   ```

5. **Сообщение `/start`** — убрать строки `commands` (текстовый список /app, /buy, /profile), оставить заголовок, буллеты, shopName, aboutText

## Risks / Trade-offs

- **Дублирование кода**: вынос тел команд увеличивает количество экспортируемых функций, но устраняет дублирование между командой и кнопкой
- **Совместимость**: текстовые команды продолжают работать как раньше — изменение только добавляет новый способ навигации
- **Callback-коллизии**: `nav:*` не пересекается с существующими `admin:*`, `buy:*`, `trial:*` и т.д.
