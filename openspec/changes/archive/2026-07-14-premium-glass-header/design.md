## Context

Хедер Mini App (`miniapp/src/App.tsx:104-117`) — это sticky `.top-bar` с логотипом и балансом кредитов. Сейчас его фон — `color-mix(in srgb, var(--bg) 82%, transparent)`, что в тёмной теме (`--bg: #0a0a0b`) даёт практически чёрную полосу. `backdrop-filter: blur(14px)` не виден, потому что фон почти непрозрачен.

В проекте есть готовая система Liquid Glass с токенами `--glass-bg-dark/light`, `--glass-border-dark/light`, но хедер их не использует.

## Goals / Non-Goals

**Goals:**
- Сделать фон хедера прозрачно-стеклянным — blur должен быть визуально заметен
- Добавить нижнюю границу для отделения от контента
- Использовать существующие glass-токены, не вводя новые
- Согласовать `.wallet-pill` с glass-стилистикой
- Сохранить совместимость с AdminSettingsBar

**Non-Goals:**
- Менять структуру JSX или создавать отдельный Header-компонент (изменения только в CSS)
- Менять поведение (sticky, z-index, логика)
- Добавлять новые элементы в хедер
- Менять нижний dock / навигацию

## Decisions

### 1. Фон хедера — glass-токены вместо color-mix

**Решение:** Заменить `color-mix(in srgb, var(--bg) 82%, transparent)` на `var(--glass-bg-dark)` / `var(--glass-bg-light)` и увеличить blur до `20px`.

**Почему:** Glass-токены уже имеют нужную прозрачность (~55%) и согласованы со всей системой. Blur 14px → 20px усилит ощущение стекла.

```css
/* Было */
background: color-mix(in srgb, var(--bg) 82%, transparent);
backdrop-filter: blur(14px);

/* Стало — тёмная тема */
background: var(--glass-bg-dark);
backdrop-filter: blur(20px);

/* Стало — светлая тема */
background: var(--glass-bg-light);
backdrop-filter: blur(20px);
```

### 2. Нижняя граница

**Решение:** Добавить `border-bottom` с использованием `--glass-border-dark` / `--glass-border-light`.

**Почему:** Хедер визуально висит в воздухе без нижней границы. Прозрачный фон усилит эту проблему — контент будет «заезжать» под хедер через blur. Граница фиксирует границу между хедером и контентом.

В светлой теме `--glass-border-light` (`rgba(255,255,255,0.6)`) не даст видимого контраста на светлом фоне — используем `rgba(0,0,0,0.08)` через переменную.

### 3. Wallet-pill — убрать border в пользу inset shadow

**Решение:** Заменить `border: 1px solid var(--hairline)` на тонкий inset glass-эффект, чтобы пилюля выглядела частью стеклянной поверхности, а не отдельным элементом.

**Почему:** На стеклянном фоне хедера border пилюли выглядит как наклейка. Без border, с мягким фоном, она смотрится интегрированной.

### 4. AdminSettingsBar — наследует стиль хедера

**Решение:** AdminSettingsBar уже использует `<GlassPanel>`, что даёт ему glass-стиль. Его позиционирование не меняется — он рендерится внутри AdminScreen, а не в хедере. Изменений не требуется.

### 5. Scroll-aware плотность (опционально)

**Решение:** Не реализуем в этом change. Если понадобится — отдельный change.

**Почему:** Добавляет complexity с IntersectionObserver / scroll-событиями. Текущая задача — визуальное качество, не интеракция.

## Risks / Trade-offs

- **[Blur performance]** `backdrop-filter: blur(20px)` тяжелее для GPU, чем 14px, особенно при скролле. → На мобильных устройствах Telegram Mini App это может быть заметно. Если появятся жалобы — снизить до 16px.
- **[Light mode border]** В светлой теме glass-border-light (белый) не виден на светлом фоне. → Используем отдельное правило для `[data-scheme="light"]` с тёмным полупрозрачным border.
- **[Content read-through]** Прозрачный хедер может накладываться на белый текст/элементы. → Blur + 55% opacity дают достаточную непрозрачность для читаемости. Проверить на контентных страницах (PromptScreen с длинным текстом).
