## Context

Текущий плеер состоит из двух поверхностей: PlayerBar (мини-плеер над доком) и PlayerScreen (полноэкранный оверлей). Оба используют Liquid-glass эстетику — blur, hairline-бордеры, accent-цвета.

Проблема композиции:
- **PlayerBar**: play-кнопка, инфо о треке и громкость — в одной flex-строке. Два слайдера (прогресс + громкость) рядом без визуального разделения.
- **PlayerScreen**: четыре зоны с равномерным `gap: 16px` — арт, инфо, прогресс, play-кнопка 72px, громкость — без группировки, иерархия размыта.
- **player.tsx**: не экспортирует `currentTime` / `duration` для time labels.

Liquid-glass сохраняется, трогаются только компоновка и иерархия.

## Goals / Non-Goals

**Goals:**
- PlayerBar: двухрядная компоновка — первый ряд play + инфо, второй ряд прогресс + громкость
- PlayerScreen: группировка play-кнопки и громкости в единый блок контролов; прогресс-бар с time labels (mm:ss / mm:ss)
- Добавить `currentTime` / `duration` в PlayerState (player.tsx)
- Обновить CSS — новые сетки, скорректировать gap, убрать лишний воздух
- Адаптивность: на узких экранах (< 360px) громкость в баре скрывается

**Non-Goals:**
- Не меняем API плеера (toggle, seek, setVolume, nextTrack и т.д.)
- Не добавляем кнопки next/previous (могут быть в будущем)
- Не меняем логику Audio-элемента
- Не трогаем Media Session API
- Не добавляем обложки (placeholder остаётся)
- Не меняем TrackPlayButton (кнопка в строках списка)

## Decisions

**1. PlayerBar: два flex-ряда вместо одного**

Текущая однорядная компоновка заставляет конкурировать play, текст и volume. Решение:

```
┌───────────────────────────────────────────┐
│ [▶]  Название трека                       │ ← .player-bar-row
│      Исполнитель                          │
├──── .player-bar-divider (hairline) ───────┤
│ [━━━━━━━━━━━━━━━]  [🔊] [━━━━━━━━━━]     │ ← .player-bar-row
└───────────────────────────────────────────┘
```

- `.player-bar` переходит на `flex-direction: column; gap: 0;`
- Внутри два `.player-bar-row` с `display: flex; align-items: center;`
- Первый ряд: `TrackPlayButton` + `player-info` (flex: 1)
- Второй ряд: `player-progress` (flex: 1) + `VolumeControl` (фикс. ширина)
- Разделитель — hairline-линия между рядами (опционально, зависит от визуального шума)
- `player-progress` расширяется с `flex: 0 0 72px` до `flex: 1` (занимает доступное место)
- Высота бара остаётся ~ той же за счёт более эффективного использования пространства

*Альтернатива:* volume на второй строке без прогресса (прогресс под play). Отклонено: два слайдера в разных строках без связи — пользователь не видит, что они относятся к разным функциям.

**2. PlayerScreen: grouped controls panel**

Текущая разряженная вертикаль заменяется на группировку:

```
┌──────────────────────────────────┐
│ ← Сейчас играет                  │
│                                  │
│        (artwork)                 │
│                                  │
│      Название трека              │
│      Исполнитель                 │
│                                  │
│  1:23  [━━━━━━━━━━━]  3:45      │ ← прогресс + time
│                                  │
│ ┌──────────────────────────────┐ │
│ │      [▶ 56px]               │ │ ← controls card
│ │  [🔊] [━━━━━━━━━━━━━━━━]    │ │
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```

- `.player-screen` сохраняет `flex-direction: column`
- `.player-screen-controls` становится карточкой-контейнером с фоном `--card`/`--canvas` и тонким border при необходимости
- Внутри: play-кнопка (уменьшается с 72px до 56px) сверху, VolumeControl снизу
- Либо: play слева, VolumeControl справа — в одной строке (более компактно)
- `.player-screen-progress-wrap` получает time labels: два `<span>` по бокам от полосы
- Gaps уменьшаются: между артом и инфо → 12px, между инфо и прогрессом → 8px, между прогрессом и controls → 16px

*Альтернатива:* play и volume в одной строке (▶ слева, 🔊━━━ справа). Более рискованный вариант: play-кнопка может конкурировать с громкостью за визуальный вес.

**3. Time labels из `currentTime` / `duration`**

Сейчас player.tsx экспортирует `progress: number` (0–1). Для time labels нужно добавить:

- `currentTime: number` (секунды, обновляется из `timeupdate`)
- `duration: number` (секунды, обновляется из `loadedmetadata` / `durationchange`)

Форматирование: `mm:ss` — делается на уровне компонента (PlayerScreen), не в контексте.

*Альтернатива:* форматировать время в context — отклонено: это presentation logic, не state.

**4. Liquid-glass сохраняется**

- Те же CSS-токены: `--glass-bg-dark`, `--glass-border-dark`, `--accent`, `--hairline`, `--radius-card`
- Те же backdrop-filter и box-shadow для .player-bar и .player-screen
- Изменения только в сетке/отступах/иерархии, не в визуальном стиле

**5. Адаптивность**

- По умолчанию: двухрядный PlayerBar с прогрессом и громкостью
- `< 360px` ширина: `.volume-slider` скрыт (как сейчас), остаётся только иконка mute-toggle
- `< 360px` ширина: время на прогресс-баре в PlayerScreen можно скрыть или уменьшить шрифт
- PlayerScreen на узких экранах: controls card растягивается на всю ширину

## Risks / Trade-offs

- **Бар становится выше**: два ряда вместо одного увеличивают высоту PlayerBar на ~20px. Для очень маленьких экранов это может быть заметно. → Mitigation: сделать второй ряд менее высоким (compact progress + volume).
- **Два слайдера в одном ряду (прогресс + volume)**: пользователь может случайно кликнуть не по тому. → Mitigation: разделить иконкой громкости и/ставить небольшой отступ между группой прогресса и группой volume.
- **grouped controls panel в PlayerScreen**: может выглядеть как лишняя вложенность. → Mitigation: тонкий border или вообще без border, только группировка через отступы (меньше gap внутри, больше gap снаружи).
- **Time labels добавляют визуальный шум**: на коротких треках (< 30 сек) время будет дёргаться каждую секунду. → Mitigation: форматировать в mm:ss, обновляется по timeupdate (уже есть).
