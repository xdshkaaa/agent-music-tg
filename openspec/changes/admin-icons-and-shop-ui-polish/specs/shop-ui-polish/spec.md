# shop-ui-polish

## ADDED Requirements

### Requirement: Package cards show readable prices
Карточка пакета в магазине SHALL показывать цену в читаемом формате: число без хвостовых нулей, пробел, код актива (например `1 TON`), и Stars-цену как `N ⭐`; кнопки цен SHALL иметь единый стиль и высоту.

#### Scenario: TON price formatted
- **WHEN** оффер имеет amount `111` и asset `TON`
- **THEN** кнопка цены показывает «111 TON» одним визуальным блоком, без слипания цифр

#### Scenario: Fractional amount trimmed
- **WHEN** amount равен `1.50`
- **THEN** кнопка показывает «1.5 TON»

### Requirement: Store search has empty state
Магазин SHALL показывать явное пустое состояние (иконка + текст), когда поиск или фильтр категории не находит ни одного пакета.

#### Scenario: No search results
- **WHEN** пользователь вводит запрос, которому не соответствует ни один пакет
- **THEN** вместо пустого списка показывается сообщение «ничего не найдено» с приглушённой иконкой

### Requirement: Downloads list uses compact controls
Список загрузок в профиле SHALL использовать компактные play-кнопки (не крупнее 32px), выровненные строки трека (иконка, название, статус-индикатор в одну линию) и одинаковые вертикальные отступы между строками.

#### Scenario: Track rows aligned
- **WHEN** пользователь раскрывает загрузку с несколькими треками
- **THEN** каждая строка трека показывает компактную play-кнопку, название и (при наличии) warning-иконку, выровненные по одной базовой линии

### Requirement: Balance block has visual hierarchy
Блок баланса в профиле SHALL визуально разделять основное значение (крупный баланс) и вторичные данные (статус подписки), с кнопкой пополнения, не доминирующей над значением баланса.

#### Scenario: Balance emphasized
- **WHEN** пользователь открывает профиль
- **THEN** значение баланса — самый крупный текст блока, статус подписки — вторичный приглушённый текст, кнопка «Пополнить» вписана в высоту блока
