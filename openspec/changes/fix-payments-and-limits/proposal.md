# Fix payments and limits

## Why

Покупки, трата генераций и подписка «на бумаге» реализованы, но end-to-end не подтверждены как рабочие: пользователь должен реально мочь купить оффер (Crypto Pay и Stars), увидеть зачисление баланса, тратить одну генерацию за успешный результат и генерировать по подписке без списаний. Кроме того, подписка сейчас безлимитна — нужен почасовой рейт-лимит, чтобы подписчик не мог выжигать LLM-бюджет без ограничений.

## What Changes

- Аудит и починка механизма покупки end-to-end: оффер → инвойс (crypto/stars) → оплата → webhook/поллер → зачисление кредитов или продление подписки → отражение в `/api/me`, магазине Mini App и боте. Все найденные разрывы (несозданный инвойс, незачисленный грант, неотображённый баланс) устраняются.
- Гарантия корректной траты генераций: ровно один кредит (trial → paid) списывается за каждую *успешную* генерацию по всем путям (`/generate`, `/generate/stream`, resume, extend, бот); clarify-раунды и ошибки бесплатны; двойное списание невозможно.
- Работоспособность подписки: активная подписка даёт доступ без списания кредитов; продление складывается с остатком; истёкшая подписка перестаёт давать доступ.
- **Новое**: почасовой рейт-лимит для подписчиков — не более N успешных генераций за скользящий час (конфиг `SUBSCRIPTION_HOURLY_LIMIT`, по умолчанию 25). При превышении — ответ «лимит исчерпан» с временем, когда можно снова. Кредитные/trial-пользователи и админы не ограничиваются. Лимит проверяется до запуска LLM.

## Capabilities

### New Capabilities

- `subscription-rate-limit`: почасовой лимит успешных генераций для пользователей, чей доступ идёт по подписке; конфигурация, проверка до старта, сообщение об исчерпании с временем восстановления.

### Modified Capabilities

- `payments`: требования на рабочий цикл покупки — создание инвойса для активного оффера (crypto и stars), идемпотентное исполнение оплаты, зачисление гранта, видимость результата покупки пользователю.
- `generation-access`: уточнение требований списания — один кредит за успешную генерацию на каждом входном пути (включая SSE-стрим), бесплатность clarify/ошибок, отсутствие списания у подписчиков, отказ в доступе при нуле средств.

## Impact

- `server/access/entitlements.ts` — точка входа проверки доступа; добавится проверка рейт-лимита подписки.
- `server/access/generations-store.ts` — подсчёт успешных генераций за последний час (по `created_at`).
- `server/core/run-generation.ts` — возврат нового исхода `rate_limited` (или расширение `needs_purchase`) до вызова LLM.
- `server/api/routes.ts` — проброс нового исхода в JSON- и SSE-ответы; `server/bot/*` — сообщение в боте.
- `server/payments/*` (`purchase.ts`, `fulfillment.ts`, `stars.ts`, `webhook.ts`, `poller.ts`) — фиксы по результатам аудита.
- `miniapp/src/screens/BuyScreen.tsx`, `PromptScreen.tsx`, `lib/api.ts` — обработка `rate_limited`, корректное отображение баланса/подписки после покупки.
- `server/env.ts` — новая переменная `SUBSCRIPTION_HOURLY_LIMIT`.
- Тесты: `server/payments/payments.test.ts`, `server/api/*.test.ts`, новые тесты рейт-лимита.
