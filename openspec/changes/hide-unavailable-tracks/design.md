## Context

После генерации плейлиста запускается фоновая проверка доступности треков (`pre-play-track-verification`). Статус каждого трека (`pending | checking | verified | unavailable`) приходит в Mini App через polling `GET /api/tracks/verify`. Сейчас битые треки показываются с красной иконкой и блокируются при воспроизведении. Задача — полностью убрать их из видимого списка результатов.

**Текущий flow (ResultsScreen.tsx):**
1. `useEffect` опрашивает `api.verifyTracks(uris)` раз в 2с, пока все треки не станут `verified`/`unavailable`
2. `verification[uri]` используется для иконки статуса и блокировки воспроизведения
3. Список рендерится как `playlist.tracks.map(...)` — все треки, включая недоступные

## Goals / Non-Goals

**Goals:**
- Скрывать треки со статусом `unavailable` из отображаемого списка
- Держать видимыми `checking` / `pending` / `verified` пока проверка идёт
- Показать сообщение, если после проверки не осталось видимых треков
- Убрать теперь мёртвый код блокировки `unavailable`

**Non-Goals:**
- Не менять серверную логику проверки (`GET /api/tracks/verify`, `Extractor.probe`)
- Не менять очередь воспроизведения и кнопку «Скачать»
- Не менять поведение Telegram bot

## Decisions

### 1. Фильтрация на уровне рендера

Добавить `visibleTracks = playlist.tracks.filter((t) => verification[t.uri] !== "unavailable")` и итерировать по `visibleTracks` вместо `playlist.tracks`. Это:
- Не требует изменения типов или серверного контракта
- Позволяет треку «исчезнуть» из списка ровно в момент, когда статус становится `unavailable`
- Сохраняет трек видимым, пока статус `pending`/`checking`/`verified`

### 2. Пустое состояние

Когда polling завершён (`done.current === true`) и `visibleTracks.length === 0` — рендерить сообщение «Все треки недоступны» вместо списка. Это не ломает layout, если вся пачка оказалась битой.

### 3. Очистка мёртвого кода

После фильтрации недоступные треки не рендерятся, поэтому:
- `handleTrackClick`: убрать ветку `if (status === "unavailable")` (недостижима)
- `TrackPlayButton onBeforePlay`: убрать проверку `verification[track.uri] === "unavailable"` (недостижима)
- `verificationIcon`: ветка `unavailable` недостижима, но функция всё ещё нужна для `checking`/`verified`
- Импорт `WarningCircle` остаётся (используется в toast и error-row)

## Risks / Trade-offs

- **[Мерцание списка]** Трек может исчезнуть из списка в процессе просмотра, когда приходит `unavailable` → приемлемо: это и есть целевое поведение, скрытие происходит плавно вместе с существующей анимацией `reveal-stagger` при следующем рендере.
- **[Рассинхрон счётчиков]** Сквозные индексы `--i` для анимации придётся брать от `visibleTracks`, а не от исходного массива — используем индекс из `.map` по `visibleTracks`.
