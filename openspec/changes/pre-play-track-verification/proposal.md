## Why

Сейчас проверка треков происходит в момент воспроизведения — пользователь нажимает «плей» и только тогда выясняется, что трек недоступен (удалён с YouTube, заблокирован по гео, возрастное ограничение). Это «казино»: неизвестно, сработает трек или нет. Нужно проверить все треки плейлиста заранее — сразу после генерации, до того как пользователь нажал плей. Битые треки должны быть помечены, чтобы пользователь видел статус до попытки воспроизведения.

## What Changes

- После генерации плейлиста запускать фоновую проверку каждого трека (доступен ли для скачивания/стриминга)
- В результатах генерации показывать статус проверки для каждого трека: проверяется, доступен, недоступен
- Пользователь видит иконку статуса рядом с треком до нажатия «плей»
- Битые треки не отправляются авто-аудио в Telegram
- Статус проверки хранится в сессии/результатах генерации
- Проверка использует существующий `YtDlpExtractor` (лёгкий пробный запрос, не полная загрузка)
